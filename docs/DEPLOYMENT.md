# Deploying Fiyox — Neon (Postgres) + Render (API) + Vercel (frontend)

All three have free tiers. Total time: ~30 minutes. Do the steps in order —
each layer needs a value from the previous one.

> Security first: your database URL, `SECRET_KEY`, and Paystack key are
> secrets. They go **only** into the Render/Vercel dashboards — never into
> git, never into chat, never into frontend code.

---

## 1. Neon — the production database (~5 min)

1. Sign up at https://neon.tech and create a project (e.g. `fiyox`), region
   closest to your users (EU works well for Nigeria).
2. On the project dashboard, copy the **connection string** — it looks like
   `postgresql://user:password@ep-xxx.aws.neon.tech/neondb?sslmode=require`.
3. That's it. Paste it **as-is** into Render in the next step — the app
   normalizes it to the async driver automatically (verified against real
   Postgres: the full migration chain and app run cleanly).

## 2. Render — the API (~10 min)

1. Sign up at https://render.com and connect your GitHub account.
2. **New + → Blueprint**, select the `fiyox-school-management-system` repo.
   Render reads `render.yaml` and proposes the `fiyox-api` web service.
3. Fill the env vars it asks for:
   - `DATABASE_URL` → the Neon string from step 1
   - `FRONTEND_ORIGIN` → leave blank for now; you'll set it after Vercel
     (temporarily `*` is acceptable to get started)
   - `PAYSTACK_SECRET_KEY`, `TERMII_API_KEY` → leave empty until you have keys
   - `SECRET_KEY` is auto-generated by Render — don't set it manually
4. Deploy. The start command runs `alembic upgrade head` **before** the
   server, so your Neon database gets the full schema on first boot and every
   future deploy migrates automatically.
5. When it's live, note your API URL, e.g. `https://fiyox-api.onrender.com`,
   and check `https://fiyox-api.onrender.com/api/health` → `{"status":"ok"}`.
6. Create your platform owner + first school: open
   `https://fiyox-api.onrender.com/docs`. There's no seed in production —
   create the super admin via a one-off Render Shell command:
   ```
   python -c "
   import asyncio
   from app.core.database import SessionLocal
   from app.core.security import hash_password
   from app.models.school import User, Role
   async def go():
       async with SessionLocal() as db:
           db.add(User(school_id=None, email='YOUR_EMAIL',
                       hashed_password=hash_password('CHOOSE_A_STRONG_PASSWORD'),
                       role=Role.SUPER_ADMIN, first_name='Daniel', last_name='Godwin'))
           await db.commit()
   asyncio.run(go())"
   ```
   Then onboard your first school through `POST /api/schools` in `/docs`.

> Free-tier note: Render free services sleep after ~15 minutes idle; the
> first request after sleep takes ~30–60s. Fine for pilots; upgrade later.

## 3. Vercel — the frontend (~10 min)

1. Sign up at https://vercel.com with GitHub and **Import** the repo.
2. Set **Root Directory** to `frontend` (critical — the Next.js app lives there).
3. Add one environment variable:
   - `NEXT_PUBLIC_API_URL` = your Render URL (e.g. `https://fiyox-api.onrender.com`)
4. Deploy. You get e.g. `https://fiyox.vercel.app`.
5. Go **back to Render** and set `FRONTEND_ORIGIN=https://fiyox.vercel.app`
   (exact URL, no trailing slash), then redeploy the API. Browsers can now
   only call your API from your frontend.

## 4. Go-live extras (when ready)

- **Paystack**: set `PAYSTACK_SECRET_KEY` on Render, and in the Paystack
  dashboard set the webhook URL to
  `https://<your-render-url>/api/fees/paystack/webhook`. "Pay online" turns on
  by itself.
- **Termii SMS**: set `TERMII_API_KEY` (+ `TERMII_SENDER_ID`) on Render;
  announcements and fee reminders start transmitting for real.
- **Custom domain** (e.g. `fiyox.ng`): add it in Vercel (frontend) and Render
  (API, e.g. `api.fiyox.ng`), update `NEXT_PUBLIC_API_URL` and
  `FRONTEND_ORIGIN` accordingly.

## 5. Post-deploy checklist

- `GET /api/health` returns ok on the Render URL
- Log in on the Vercel URL as your super admin, onboard a school
- School admin logs in → creates a teacher via **Users** → teacher enters
  scores → admin computes + publishes → parent account sees the report
- CI stays the release gate: only push to `main` when the badge is green
